Minimundo: Sistema de Gestão de Distribuição e Rastreabilidade de Sementes
1. Contexto Geral O sistema tem como objetivo gerenciar o ciclo de vida da doação e distribuição de sementes para agricultores, garantindo o controle de estoque, a rastreabilidade do insumo e o acompanhamento técnico do plantio no campo. O sistema deve funcionar como uma ferramenta de apoio para técnicos agrícolas e administradores.
2. Gerenciamento de Atores (Agricultores e Técnicos) O sistema deve manter um cadastro detalhado dos Agricultores beneficiados, armazenando informações pessoais como nome, CPF, endereço, telefone, e obrigatoriamente o número de registro no IPA (Instituto Agronômico de Pernambuco ou similar), além da data de cadastro. Além disso, o sistema gerencia os Técnicos responsáveis pelas visitas e distribuições. Para cada técnico, é necessário registrar nome, e-mail, telefone e a área pela qual ele é responsável.
3. Gestão de Sementes e Rastreabilidade O controle de estoque é feito através da entidade Semente. O sistema deve registrar o nome da semente, tipo de cultura, número do lote, data de validade, origem e a quantidade disponível em estoque. Para garantir a segurança alimentar e logística, existe um módulo de Rastreabilidade, que registra toda a movimentação das sementes (origem, destino, data da movimentação e tipo de movimento), permitindo saber exatamente o histórico de cada lote.
4. Processo de Distribuição A função central do sistema é registrar as Distribuições. Uma distribuição ocorre quando uma quantidade específica de sementes é entregue a um agricultor. O registro deve conter a data da entrega, quem foi o responsável pela entrega, a quantidade entregue e o status de sincronização (indicando se o dado foi coletado offline e enviado ao servidor).
5. Monitoramento de Cultivo (Pós-Distribuição) Após a semente ser distribuída, o sistema deve permitir o acompanhamento do seu desenvolvimento através da entidade Cultivos. Os técnicos podem registrar o progresso da lavoura vinculada a uma distribuição específica. Esses registros incluem a fase de crescimento (ex: germinação, floração), uma descrição textual do estado da plantação, a data do registro e, opcionalmente, uma foto da lavoura ("foto lavoura") para evidência visual.
6. Controle de Acesso e Auditoria Para acessar o sistema, existem Usuários com credenciais (e-mail e senha criptografada) e níveis de permissão (tipo de usuário). O sistema possui mecanismos de segurança robustos:
Sincronizações: Como o sistema pode ser utilizado em áreas com baixa conectividade, há um registro de logs de sincronização (tabela afetada, data e status) para garantir a integridade dos dados transmitidos.
Histórico de Registros (Auditoria): Todas as ações críticas no sistema geram um log de auditoria, registrando qual usuário realizou a ação, qual tabela foi alterada, o tipo de ação (inserção, alteração, exclusão) e a data exata, garantindo total transparência operacional.
